# 原始输入

现在前端的看板页面需要优化处理，需要体现出效能，比如当前任务从创建到更新为归档花费的时间和之前的时间进行对比，提升百分之多少，那种具有实际增量意义的图表。

请分析如何优化，当前的图表可以折叠为一行或独立一个模块，比如新菜单处理上述的新表格，保留当前表格菜单。

用户确认：

- 多菜单方案是对的，就这样处理。
- 没有老数据作为对比时，可以从之前访谈中信息推测：产品说过“新的需求以前要一两，现在 2 天就弄完了，也几乎没有了研发的对文档认知的返工”，大概有 30%左右的提效。
- 当前数据可以从 DirectUS 接口捞。
- issue 表字段为 `created_at` 和 `updated_at`，`updated_at` 可以当做最后一个状态的变更时间。
