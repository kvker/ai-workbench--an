# 效能看板需求文档

## 背景与目标

当前报表页主要展示使用覆盖和阶段分布，缺少“AI Native 流程是否带来实际效能增量”的表达。新增效能看板，用 DirectUS 归档需求数据计算当前归档周期，并在缺少老数据时使用用户提供的访谈基线展示提效百分比。

## 范围

### 包含

- 前端新增效能看板视图并合并到报表页。
- 扩展 DirectUS 报表数据读取。
- 当前报表页布局优化。
- 文档和临时口径同步。

### 不包含

- 不新增后端接口。
- 不接入真实状态变更历史。
- 不启动本地常驻服务。

## 依赖

- DirectUS `ops_issue` 数据。
- 用户确认 issue 表 `updated_at` 可作为最后状态变更时间。
- 用户确认 `harness_archive_status = 2` 才表示归档完成。
- 用户确认开发者账号 `zhaoyue` 不参与评估统计。
- 用户要求按 P85 排序，并在 P50 右侧增加平均值。
- 用户要求不只看归档周期，还要根据 PM、FE、BE、QA、归档各阶段 status 多维展示进度。
- 用户要求将“归档周期趋势”改为“各阶段周期趋势”，将“归档需求明细”改为“各阶段需求明细”。
- 用户要求移除效能页顶部说明模块，整体宽度使用父级 100%，不要横向溢出。
- 用户要求阶段进度分布使用红黄绿三色，更紧凑展示，并将右侧需求数量统一放到标题区域。
- 用户根据截图反馈：右侧卡片看不到、左右仍溢出；红黄绿语义应表示开始、进行、结束，且颜色不要纯色。
- 用户反馈“各阶段需求明细”只有两条数据；原因是此前误用了归档完成数据源，需改为全量 Harness 需求明细。
- 用户要求基于最新代码，将 `reports` 和 `efficiency` 合并成一个 `reports`。
- 用户要求效能看板作为默认视图，使用看板放右侧，整体根目录默认进入 `reports`。
- 用户提供访谈基线：约 30% 提效。
