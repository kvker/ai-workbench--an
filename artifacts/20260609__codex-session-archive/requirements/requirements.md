# Codex 会话归档区需求文档

## 背景与目标

当前历史对话区通过双击会话卡片修改别名，操作隐蔽且容易与选择会话冲突。用户希望显式提供改名和归档操作，并把归档会话放到独立区域中支持恢复。

## 验收标准 (AC)

- [x] AC1: 活跃会话列表中的每个会话有“修改名字”按钮。
- [x] AC2: 删除双击改名逻辑和相关延迟点击代码，单击仍用于选择会话。
- [x] AC3: 活跃会话列表中的每个会话有“归档”按钮，点击后出现二次确认。
- [x] AC4: 归档后会话从活跃历史列表移到归档区。
- [x] AC5: 归档区位于会话区下方，最大高度为会话侧栏的 50%，内容可滚动。
- [x] AC6: 归档区会话可点击恢复到活跃历史列表。
- [x] AC7: 当前选中的会话被归档后，前端应自动切换到可用的活跃会话或创建新会话，避免停留在已归档会话。

## 范围

### 包含

- 前端历史对话区列表交互调整。
- service 会话 metadata 归档/恢复接口。
- 前端 Codex service 客户端新增归档状态更新方法。

### 不包含

- 删除会话。
- 批量归档/批量恢复。
- 归档数据迁移；已有会话默认未归档。
