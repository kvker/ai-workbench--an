# Service 钉钉消息发送抽象需求

## 背景与目标

参考已有钉钉发送摘要，在当前 `repos/service` 中构建可复用的钉钉消息发送抽象。根据用户打断后的最新要求，本轮只实现“能发送”的底层能力，不迁移参考项目里的审批业务模型。

## 验收标准

- [x] AC1: service 提供可复用的钉钉企业机器人单聊发送函数，调用方可以传入钉钉 userId 和 Markdown 消息。
- [x] AC2: service 提供按手机号列表发送钉钉消息的 HTTP 接口。
- [x] AC3: 实现不绑定审批实例、审批任务或参考项目的业务字段。
- [x] AC4: 缺少的钉钉配置和接收人 userId 物料已整理记录。
- [x] AC5: service 代码通过一次性语法验证。

## 范围

### 包含

- 钉钉 access token 获取与缓存。
- 钉钉企业机器人单聊批量发送。
- 开发环境消息拦截。
- 按手机号列表发送 HTTP 接口。

### 不包含

- 审批业务通知封装。
- 用户钉钉绑定 API。
- 审批状态变更触发点。

## 依赖

- 钉钉开放平台应用配置。
- 钉钉企业机器人配置。
- 可用于联调的钉钉 userId。
