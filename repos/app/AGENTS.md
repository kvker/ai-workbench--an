# AGENTS.md

本文件写给 Codex。当前目录是 AI Native 工作台的 Web 用户操作端。

## 当前项目

- 技术栈：React 19、TypeScript、Vite。
- 包管理：npm，依赖锁定在 `package-lock.json`。
- 常用命令：
  - `npm run dev`
  - `npm run build`
  - `npm run lint`
- 主要代码：
  - `src/App.tsx`
  - `src/App.css`
  - `src/index.css`
  - `src/main.tsx`

## 产品理解

这个前端是工作台的用户界面，不是完整后端系统。

工作台主路径：

```text
登录页
  -> 需求页
  -> 详情页
      -> 信息区
      -> 流程区
      -> 对话区
      -> 打开代码页
```

核心对象：

- 登录页：进入工作台前的身份校验入口。
- 需求页：展示需求列表，选择需求进入详情页。
- 详情页：单个需求的执行空间。
- 信息区：详情页内展示需求名称、整体状态、单行描述和完整详情入口。
- 流程区：详情页内展示需求分析、原型编写、文档移交、交付等流程节点。
- 对话区：详情页内承载 AI 协作会话，体验上接近在线版 Codex IM。
- 代码页：详情页顶部按钮打开的新标签页，由 vscode-server 承载。

## 架构边界

实现时遵守这些边界：

- 不要把需求页做成 AI 对话页。需求页只负责列表、筛选、识别和进入详情页。
- 不要把详情页做成普通详情展示页。详情页是当前需求的执行空间。
- 不要在信息区承载完整需求编辑。信息区只展示摘要和打开详情。
- 不要把流程区做成纯装饰步骤条。流程区需要体现节点状态、前置依赖和文档入口。
- 不要在当前 React 应用里重做 vscode-server。代码页是外部在线编辑器入口。
- 不要在当前前端中实现 Codex 内核、工具执行、Git 分支管理或工作区调度。前端只表达入口、状态和交互。
- 不要跨需求混合状态。一个详情页只绑定一个需求、一个工程文件夹和一个需求分支。

## 底层模型

参考架构采用：

```text
模板仓库
  -> 需求分支
  -> 用户工作区
```

- 一个需求对应一个需求分支。
- 多个用户可以共同维护同一个需求分支。
- 每个用户在服务器上使用不同实体文件夹作为工作区。
- 用户私有配置、缓存、临时文件、角色差异化 skills 不应进入共享需求上下文。

这些是产品与架构语义。除非任务明确要求，不要在当前 Web 端提前实现真实 Git 或文件系统逻辑。

## UI 实现原则

- 这是产研工作台，界面应简洁、克制、可扫读，偏工具产品而非营销页。
- 首页应直接呈现可操作工作台，不要做品牌落地页或大段介绍页。
- 避免装饰性过强的卡片堆叠、巨大 hero、渐变背景和无业务含义的视觉元素。
- 优先展示当前需求、流程节点、对话状态、代码页入口等用户要操作的信息。
- 移动端和桌面端都要避免文字溢出、控件挤压和内容重叠。

## 编码约定

- 优先沿用现有 Vite + React + CSS 文件结构。
- 小步修改，避免无关重构。
- 组件、状态和类型命名应贴近业务语义，例如 `Demand`, `WorkflowNode`, `DemandDetail`。
- 对固定枚举使用明确的字符串联合类型或常量，不要散落魔法字符串。
- 静态 mock 数据可以用于搭建界面，但应和架构语义一致。
- 不引入新的 UI 框架，除非用户明确要求或现有实现已经采用。

## 验证

修改代码后优先运行：

```bash
npm run build
npm run lint
```

如果改动了可视页面，启动开发服务并用浏览器检查主要视口：

```bash
npm run dev
```

不要提交或还原用户未要求处理的文件变更。
