# AI Native 工作台 Web 端

这是 AI Native 工作台的用户操作端。它面向产品、研发、测试等产研角色，提供从需求入口到详情执行空间的 Web 界面。

当前项目是一个 React + TypeScript + Vite 前端应用。它只负责用户可见的工作台交互，不直接承担账号体系、Git 分支管理、vscode-server 启动、Codex 内核或后端调度职责。

## 产品定位

工作台围绕“一个需求”组织协作。

用户先进入登录页完成身份校验，再进入需求页查看需求列表。选择某个需求后进入详情页，详情页承载该需求的后续处理流程、AI 对话、文档入口和代码页入口。

核心页面关系：

```text
登录页
  -> 需求页
  -> 详情页
      -> 信息区
      -> 流程区
      -> 对话区
      -> 打开代码页
```

## 页面边界

### 登录页

登录页只负责进入工作台前的身份校验。具体对接哪个账号体系不在当前 Web 端中绑定，落地时接入已有认证能力。

### 需求页

需求页展示需求列表，并让用户选择一个具体需求进入详情页。

入口卡片建议展示：

- 需求名
- 需求描述
- 需求状态
- 创建时间
- 更新时间
- 需求来源、优先级、负责人等辅助字段

需求页不承载 AI 对话，也不处理原始需求整理流程。

### 详情页

详情页是单个需求的执行空间。它对应底层的一个工程文件夹，并基于需求元信息渲染可操作界面。

详情页主要包含：

- 信息区：展示需求名称、整体状态、单行描述和完整详情入口。
- 流程区：展示需求分析、原型编写、文档移交、交付等节点进度。
- 对话区：承载围绕当前需求的 AI 协作会话。
- 代码页入口：在新标签页打开当前需求绑定的 vscode-server。

详情页只处理当前需求，不跨需求混合上下文。

### 代码页

代码页不是当前 React 应用内重做的编辑器，而是详情页打开的 vscode-server 在线编辑器。

它负责查看和编辑当前需求对应工程文件夹中的代码、文档、配置和交付物。Web 端只提供入口、状态提示和上下文关联。

## 协作模型

底层设计采用“模板仓库 + 需求分支 + 用户工作区”的模型。

```text
模板仓库
  -> 创建需求分支
  -> 一个分支代表一个需求
  -> 多个用户共同维护同一个需求分支
  -> 每个用户在服务器上有自己的实体工作区
```

一个需求可以由多人共同维护同一个需求分支，但每个用户使用不同的实体文件夹作为工作区，以隔离本地配置、缓存、临时文件和个人 AI 技能。

## 本地开发

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

构建生产产物：

```bash
npm run build
```

运行代码检查：

```bash
npm run lint
```

## 目录概览

```text
src/
  App.tsx       # 当前应用入口组件
  App.css       # 应用级样式
  index.css     # 全局样式与主题变量
  main.tsx      # React 挂载入口
public/         # 静态资源
```

## 设计原则

- 保持页面职责清晰：需求页做入口，详情页做执行空间，对话区做 AI 协作，代码页由 vscode-server 承载。
- 当前 Web 端优先表达用户工作流，不提前实现后端调度、分支创建、工作区分配等基础设施细节。
- 需求、流程、对话和代码上下文都应围绕“当前需求”收敛，避免多个需求混用同一页面状态。
- UI 应简洁、可扫读、偏工具属性，避免营销化或装饰性过强的表达。
