# 前端页面与区域命名规范

## 页面

| 名词 | 代码入口 | 说明 |
|------|----------|------|
| 需求页 | `repos/app/src/pages/DemandBoardPage.tsx` | 展示需求列表、筛选入口和创建需求入口。 |
| 详情页 | `repos/app/src/pages/DemandDetailPage.tsx` | 单个需求的执行空间，绑定路由参数中的需求 id。 |

## 详情页区域

| 名词 | 代码组件 | 位置 | 说明 |
|------|----------|------|------|
| 信息区 | `DemandInfoRegion` | 详情页左上角 | 只展示需求名称和查看详情入口。 |
| 流程区 | `WorkflowRegion` | 详情页左侧中部 | 展示需求分析、原型编写、文档移交、交付等流程节点。 |
| 产物区 | `ArtifactRegion` | 详情页左侧下部 | 展示当前需求相关文档和产物入口。 |
| 对话区 | `ConversationRegion` | 详情页右侧主区域 | 承载当前需求的消息列表、输入区和历史对话区。 |
| 历史对话区 | `ConversationHistoryRegion` | 对话区右侧 | 展示当前需求下的历史会话列表。 |

## 代码注释约定

前端页面和核心区域需要保留定位注释，方便后续 AI 快速匹配用户描述：

```ts
// Page: 需求页
// Page: 详情页
// Region: 信息区
// Region: 流程区
// Region: 产物区
// Region: 对话区
// Region: 历史对话区
```

## 维护要求

- 用户提到上述中文名词时，应优先定位到对应组件。
- 调整区域布局时，应同步更新本规范。
- 不要把详情页区域逻辑混回页面入口组件，页面入口只负责装配和数据读取。
- 新增或调整页面区域内的交互控件时，优先使用 antd 组件；Tailwind CSS 只用于区域布局、间距、尺寸和少量辅助 class。
