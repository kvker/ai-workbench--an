# 运行命令长期约束

## 不主动启动项目工程

- 分类：约束
- 来源：用户在 2026-06-04 明确要求后续操作不要主动启动项目工程。
- 规则：后续任务中不要主动运行项目工程或本地服务，包括但不限于 `npm run dev`、`vite`、`node server`、`nodemon`、`pm2 start` 等会常驻占用端口的命令。
- 允许：可以运行一次性验证命令，例如 build、lint、typecheck、test、格式检查、脚本扫描等，前提是该命令不会启动常驻服务或占用端口。
- 例外：只有用户本轮明确要求启动、重启或打开本地服务时，才可以执行对应运行命令。
- 行为：需要前端或接口运行态验证时，默认认为用户已经启动必要服务；优先使用已运行的服务进行验证，若无法验证则说明未运行原因，不自行启动。
